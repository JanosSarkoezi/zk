# MS-SQL Trigger abfragen und verwalten

Um alle Trigger aus einer MS-SQL Datenbank abzufragen, ist der einfachste Weg
die Nutzung der System-Views. Je nachdem, ob du nur eine einfache Liste oder
detaillierte Informationen (wie den Code des Triggers) benötigst, gibt es
verschiedene Ansätze.

Hier sind die effizientesten Methoden:

### 1. Die schnelle Übersicht (Tabellen-Trigger)

Wenn du eine Liste aller Trigger suchst, die auf Tabellen basieren, ist die
View `sys.triggers` die beste Anlaufstelle.

```sql
SELECT 
    t.name AS TriggerName,
    parent.name AS TabellenName,
    t.is_disabled AS IsDisabled,
    t.create_date AS ErstelltAm
FROM sys.triggers t
INNER JOIN sys.objects parent ON t.parent_id = parent.object_id
ORDER BY TabellenName;

```

### 2. Detaillierte Liste inkl. Schema und Typ

Oft möchte man wissen, ob es sich um einen `INSERT`-, `UPDATE`- oder
`DELETE`-Trigger handelt. Hier hilft `sys.objects`:

```sql
SELECT 
    SCHEMA_NAME(o.schema_id) AS SchemaName,
    o.name AS TriggerName,
    OBJECT_NAME(o.parent_object_id) AS ParentName,
    type_desc AS Typ,
    create_date
FROM sys.objects o
WHERE type = 'TR';

```

### 3. Trigger inklusive SQL-Code auslesen

Wenn du den tatsächlichen Programmcode der Trigger sehen möchtest, musst du
`sys.sql_modules` einbeziehen:

```sql
SELECT 
    OBJECT_NAME(m.object_id) AS TriggerName,
    m.definition AS TriggerCode
FROM sys.sql_modules m
INNER JOIN sys.objects o ON m.object_id = o.object_id
WHERE o.type = 'TR';

```

---

### Wichtige Hinweise für die Praxis

* **DDL-Trigger:** Die oben genannten Abfragen zeigen nur **DML-Trigger** (Data
  Manipulation Language, also auf Tabellen). Wenn du Trigger suchst, die auf
  Datenbank-Ebene reagieren (z. B. beim Erstellen von Tabellen), musst du
  `sys.server_triggers` oder `sys.database_triggers` abfragen.
* **Deaktivierte Trigger:** Achte in der ersten Abfrage auf die Spalte
  `is_disabled`. Ein Trigger kann existieren, aber keinen Effekt haben, wenn er
  abgeschaltet wurde.

> **Tipp:** Wenn du die Trigger nur kurz prüfen willst, ohne SQL zu schreiben,
> findest du sie im **SQL Server Management Studio (SSMS)** im Object Explorer
> unter: `DeineDatenbank` > `Tables` > `DeineTabelle` > `Triggers`.
